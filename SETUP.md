# Сертификаты (ШАҲОДАТНОМА) — приложение учёта, печати и реестра

Веб-приложение для создания, печати и учёта сертификатов Тоҷикстандарт. Заменяет ручное
заполнение в Word + перенос в Excel одной программой.

## Возможности

- **Новый сертификат** — шаблон бланка формата **A4** с постоянным текстом и редактируемыми полями
  (rich text editor на базе **Lexical**). Пользователь заполняет выделенные поля; при необходимости
  можно исправлять и сам постоянный текст (правки шаблона сохраняются локально в браузере и
  применяются ко всем сертификатам; кнопка «Сбросить текст шаблона» возвращает исходный текст).
- **Печать** — печатается **только чёрный текст** поверх готового бумажного бланка (шапка, логотипы,
  рамка и номер уже напечатаны на бумаге). Печать A4, без интерфейса, с калибровкой координат.
- **Реестр** — Excel-подобная таблица на **AG Grid Community**: фильтры, сортировка, поиск,
  редактирование ячеек, изменение ширины/перемещение/закрепление колонок, экспорт CSV, итоги.
- **Хранение** — все записи в **Supabase** (`created_at`, `updated_at`, `id uuid`).

## Стек

Next.js 14 (App Router) · React 18 · TypeScript · Tailwind · Lexical · AG Grid Community ·
Supabase JS. Деплой — Vercel.

## Переменные окружения

Создайте `.env.local` (уже в `.gitignore`) на основе `.env.example`:

```
NEXT_PUBLIC_SUPABASE_URL=https://ВАШ-ПРОЕКТ.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш-публичный-anon/publishable-ключ
```

> ⚠️ На клиенте используется **только публичный** ключ. `service_role` / secret ключ **никогда**
> не помещайте в код или переменные `NEXT_PUBLIC_*`.

## База данных Supabase

Выполните один раз в **Supabase → SQL Editor** скрипт [`supabase/schema.sql`](supabase/schema.sql).
Он создаёт таблицу `certificates` (все поля сертификата + доп. данные), триггер `updated_at` и
открытые RLS-политики для публичного ключа.

Структура таблицы (кратко): `id uuid pk`, поля бланка (`cert_number`, `from_day/month/year`,
`to_day/month/year`, `service_name`, `address`, `manager_name`, `service_type`, `normative_doc`,
`basis_date_number`, `activity_doc`, `inspection_body`, `special_notes`, `signatory`), доп. данные
(`application_number`, `plan_number`, `inspector`, `amount`), `created_at`, `updated_at`.

## Запуск локально

```bash
npm install
npm run dev
```

Откройте http://localhost:3000 — по умолчанию открывается «Новый сертификат».

Production-сборка:

```bash
npm run build
npm run start
```

## Проверка сохранения

1. Заполните поля сертификата и «Дополнительные данные».
2. Нажмите **«Сохранить в реестр»** → появится уведомление об успехе.
3. Откройте **«Реестр»** — запись появится в таблице. То же видно в Supabase → Table editor.
4. Отредактируйте ячейку в реестре — изменение сохранится в Supabase (проверяется перезагрузкой).

## Проверка печати

1. Нажмите **«Печать сертификата»** (или Ctrl+P).
2. В предпросмотре должна быть страница A4 **только с чёрным текстом**, без интерфейса, подсветки и
   подчёркиваний полей.
3. Если при печати на физический бланк текст смещён — подправьте координаты в
   [`src/styles/print.css`](src/styles/print.css): переменные `--cal-x`, `--cal-y` (глобальное
   смещение в мм), а также `--cert-top` / `--cert-left` (старт блока текста).

## Деплой на Vercel

1. Запушьте репозиторий на GitHub.
2. На https://vercel.com → **New Project** → импортируйте репозиторий.
3. В **Environment Variables** добавьте `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. **Deploy**. Next.js собирается на Vercel без дополнительной настройки.

## Соответствие полей реестру (Excel)

| Колонка реестра | Поле |
|---|---|
| № свидетельства | `cert_number` |
| № заявка | `application_number` (доп. данные) |
| Наименование … получивших свидетел. | `service_name` |
| Адрес | `address` |
| ФИО предприниматель | `manager_name` |
| Вид услуга | `service_type` |
| № плана | `plan_number` (доп. данные) |
| Дата выдачи | вычисляется из даты «Эътибор дорад аз» |
| Инспектор | `inspector` (доп. данные) |
| Сумма | `amount` (доп. данные) |
